# Button

> A primitive for invoking actions. They should not be used to navigate to new pages or anchors on the current page - use
> a [Link](?path=/docs/typography-link--documentation) in these scenarios instead.

## Example

<Source dark />

## Controlling level

The `level` prop controls the button appearance. There are four available levels, defined as `primary`, `secondary`,
`tertiary` and `danger`.

### Primary

> The `primary` level should be used as the main call to action within a page or button group.

<Canvas>
    <Story of={ButtonStories.PrimaryMedium} />
</Canvas>

### Secondary

> The `secondary` level should be used as a secondary call to action within a page or button group.

<Canvas>
    <Story of={ButtonStories.SecondaryMedium} />
</Canvas>

### Tertiary

> The `tertiary` level should be used as any button that is not a call to action or destructive.

<Canvas>
    <Story of={ButtonStories.TertiaryMedium} />
</Canvas>

### Danger

> The `danger` level should be used for triggering any destructive action.

<Canvas>
    <Story of={ButtonStories.DangerMedium} />
</Canvas>

## Controlling size

The `size` prop controls button size. There are three available sizes, defined as `small`, `medium` and `large`.

## Block mode

> The button is by default set to `inline-flex` and will be sized according to its contents. On small screens or where
> only one option is available to the user, e.g. within a modal, then we may use the `block` prop to set it to full width.
